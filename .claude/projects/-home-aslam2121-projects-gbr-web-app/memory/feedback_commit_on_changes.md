Always commit changes after any feature is added or code is modified. Also create a markdown file in `docs/commits/` documenting what was done in the commit.

**Why:** User explicitly requested this workflow — they want a clear commit trail with accompanying documentation.

**How to apply:** After completing any task that modifies code, create a numbered markdown file in `docs/commits/` summarizing the changes, then stage and commit everything together.
